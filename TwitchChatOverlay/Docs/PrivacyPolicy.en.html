TwitchChatOverlay Privacy Policy

This Privacy Policy explains what information TwitchChatOverlay accesses, how it is stored and used, and how users can revoke consent or delete locally stored authorization data when the app uses Twitch and YouTube API Services. Please review this policy before using the YouTube feature.

1. About the App

TwitchChatOverlay is a Windows desktop application that displays Twitch and YouTube live event notifications and live chat notifications for a broadcaster. The YouTube feature uses YouTube API Services.

2. Information Accessed or Stored

For the YouTube feature, the app handles the following information:

  • YouTube OAuth access token
  • YouTube OAuth refresh token
  • Authorization status text including token acquisition time
  • Display name, message text, Super Chat information, and membership information received from YouTube Live Chat

Live chat display data is processed in memory to show notifications. The app is not intended to permanently store YouTube user data or chat history beyond locally stored OAuth-related information.

3. Purpose of Use

  • To display YouTube live chat, Super Chat, and membership notifications
  • To detect stream start and maintain the live chat connection
  • To allow the user to continue using the app without re-authorizing every time

The app does not use YouTube data for advertising, sale, third-party transfer, profiling, or sale of derived metrics.

4. Storage and Protection

YouTube OAuth tokens and app settings are stored in a local settings file after being protected with Windows user-scoped data protection. The storage location is typically the TwitchChatOverlay settings file under the user's Application Data folder.

The app requests only a YouTube read-only OAuth scope.

5. Third-Party Sharing

The app is not intended to send YouTube OAuth tokens or settings to any developer-operated external server or sell them to third parties. Communication with YouTube API Services, Google pages, or YouTube pages explicitly opened by the user is handled directly by the respective services.

6. Revocation and Deletion

Users can manage stored data and access in the following ways:

  • Delete locally stored YouTube authorization data from the YouTube tab in the app
  • Attempt Google-side revocation from the YouTube tab in the app
  • Manually revoke access to the app from the Google permissions management page

If the user revokes Google-side access and also wants locally stored data removed, they should perform the local deletion action in the app as well. Local deletion does not delete data stored by YouTube.
